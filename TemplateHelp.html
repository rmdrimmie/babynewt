1142 Template Tag1142 Template Tag Action
[$NAME$]This is your username that you are logged in with.
[$UNUMBER$]This is your usernumber
[$TAGLINE$]This 1142 templating tag will be replaced in the output with a randomly select tagline from the database. Only one unique random tagline will be displayed per page, no matter how many templating tags of this type are present.
[$STYLESHEET$]This tag will output the user defined stylesheet as stored in the database and selected by the user using this template. There is no need to wrap this tag within <STYLE></STYLE> HTML tags.
[$OPTIONCATLIST$]This will output the list of available categories wrapped in <OPTION></OPTION> HTML tags. This templating tag should be surrounded by <SELECT></SELECT> HTML tags to allow custom form generation.
[$POSTBOX$]This will add the comment Posting box to the page, it should be wrapped with <form name="frmComment" action="post.php" method="post"> </FORM> for the time being and include the submit button.
[$COMMENTBUTTON$]This will output a submit button with the comment number as the label, and the class as CategoryCMTBTN. The remainder of the form is still required to be entered manually.
[$CMTBTNFORM$]This will output a complete form with submit button with the comment number as the label, and the class as CategoryCMTBTN.
[$CATNAMELINK$]This templating tag is only valid within the Comment section. It will output the category name with or without a link to changecategory.php depending on the page viewer.
[$CATCSSNAME$]Each category of 1142 has a corresponding CSS name that is prepended to the different scetions of a comment for super control over display styles.
[$USERNAME$]This templating tag is on valid within the Comment section, and will output the username of the EFT that posted the comment.
[$USERNUMBER$]This templating tag is on valid within the Comment section, and will output the usernumer of the EFT that posted the comment.
[$COMMENTTEXT$]Only valid within the Comment section, this is the meat of 1142. (Or corresponding vegetarian staple).
[$CATEGORYNAME$]The visual name of the category, this changes week to week to confuse new EFTs. Use it in the Comment section of your template to display the category the of the post.
[$COMMENTDATETIME$]What else do you want? Its the date and time of the comment. Only in the comment section please.
[$CURDATE$]Current 1142 date in MM.DD.YY format. Sorry timezone is still EftST.
[$CURTIME$]Current 1142 time in HH.MM.SS format
[$BOOKMARK$]If you add in a form with prev and next buttons, you'll need a bookmark also.
[$PRESENT$]Adds a link to the most recent post.
[$15POSTPREFIX=""$]This text will be output immediately before each username in the 15POSTERS list. Must be assigned in the Template Header section.
[$15POSTSUFFIX=""$]This text will be output immediately after each username in the 15POSTERS list. Must be assigned in the Template Header section.
[$15VISITPREFIX=""$]This text will be output immediately before each username in the 15LURKERS list. Must be assigned in the Template Header section.
[$15VISITSUFFIX=""$]This text will be output immediately after each username in the 15LURKERS list. Must be assigned in the Template Header section.
[$15POSTERS$]Shows a list of users who have posted in the last 15 minutes.
[$15LURKERS$]Shows a list of users who have visited in the last 15 minutes.
  
[$DATEFORMAT=""$]This will allow you to format the date/time output of the comment. Youu still use the [$COMMENTDATETIME$] tag, but add this tag to your header section. The date format must be enclosed in quotes and it must be in the header section of the template. If it isn't you will see an error (and no posts) on your page, as will everyone else using your template. The date/time format codes are listed below. For example 
[$DATEFORMAT="%D %y %a %d %m %b %j"$]
will output the date like this: 
4th 97 Sat 04 10 Oct 277
[$GMTOFFSET=x$]This will allow you to alter the hour value of the comment timestamp by x. For example, -4 is Eastern during Daylight Savings Times, -5 is during normal times.
[$PRESENTLINKSTR="Present"$]Formats the present link text, must be in header, enclose text in qoutes
[$LOGOUTLINKSTR="Logout"$]Formats the logout link text, must be in header, enclose text in quotes

Notes:
  • To add in a category filter you need a form with an element named 'iFilterCategory' that will supply the category ([$OPTIONCATLIST$] works good because you supply the surrounding <SELECT> tags). If the filter is to be added to the same form as one containing the navigation buttons, you will also need an element named 'fltAp'. A checkbox works good for this. Example: 
    	<form name="frmFilter" action="main.php" method="post">
        [$BOOKMARK$]
	Jump to post: <input type="text" name="ViewPost" class="PageSize" value="" size="5"><BR>
	<input type="submit" name="btnPrevPage" class="PrevPage" value="Previous Page"> 
	<input type="text" name="txtPageSize" class="PageSize" value="50" size="2">
	<input type="submit" name="btnNextPage" class="NextPage" value="Next Page">
 	<LABEL ACCESSKEY=G><input type="submit" name="btnSubmitFilter" value="Go"></LABEL><BR>
        Filter: <select name="iFilterCategory" class="selectCategory">
          [$OPTIONCATLIST$]
        </select><BR>
        <INPUT TYPE="checkbox" NAME="fltAp"> Apply Filter
      </form>
    This form has: page forward &mark page back buttons, a check box to enable filtering, a category list for filtering, a spot to specify the number of posts to display when using the form, and a go button (applies filter without change location)
